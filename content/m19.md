## 📚 Теория

### Уровни зрелости ML-систем

0. Ноутбук + ручной запуск («у нас сейчас»).
1. Модель как сервис: API, контейнер, CI/CD, версии.
2. Автоматизированный пайплайн: переобучение по расписанию/триггеру, валидационные гейты.
3. Полный MLOps: feature store, реестр моделей, автоматический откат, A/B на моделях.

Большинству команд достаточно уровня 1–2. Не стройте уровень 3 ради галочки.

### Модель как артефакт

- **Сериализация:** joblib/pickle (⚠️ версии библиотек должны совпадать!), ONNX — переносимый формат.
- **Версионирование:** модель + код + данные + метрики фиксируются вместе (MLflow Model Registry, W&B). Вопрос «какая модель сейчас в проде и на чём она обучена?» должен иметь ответ за 10 секунд.
- **Reproducibility:** зафиксированные random_state, версии пакетов (lock-файл), код в git.

### Три слоя мониторинга ML-сервиса

1. **Сервисный (как у любого бэкенда):** латентность p50/p99, RPS, ошибки 5xx.
2. **Дрейф данных (data drift):** распределение входных признаков уехало от обучающего (новый сегмент клиентов, сломался источник поля). Метрики: PSI, KS-тест по каждому признаку. **Дрейф предсказаний** — сдвиг распределения скоров — самый дешёвый ранний сигнал.
3. **Качество (concept drift):** истинные метки приходят с задержкой (узнаём об оттоке через 30 дней) → считаем отложенное качество, сравниваем с офлайн-оценкой.

Правило: **модель без мониторинга — это инцидент, который ещё не случился.** Данные меняются всегда.

### Стратегии выкладки моделей

- **Shadow mode:** новая модель предсказывает параллельно, ответы логируются, но не используются. Бесплатная проверка на реальном трафике.
- **Canary:** 5% трафика на новую модель, метрики ок → раскатка.
- **Откат:** предыдущая версия всегда наготове; переключение — конфигом, не деплоем.

## 💻 Практика

### FastAPI-сервис

```python
# app.py
from fastapi import FastAPI
from pydantic import BaseModel, Field
import joblib

app = FastAPI(title="churn-model", version="2.3.0")
model = joblib.load("model/churn_lgbm_v2.3.joblib")

class Features(BaseModel):
    tenure_months: float = Field(ge=0)
    monthly_fee: float
    logins_last30: int = Field(ge=0)
    support_tickets: int
    negative_tickets: int
    days_since_last_login: float
    autopay_enabled: int

@app.post("/predict")
def predict(f: Features):
    proba = model.predict_proba([[*f.model_dump().values()]])[0, 1]
    return {"churn_probability": round(float(proba), 4),
            "model_version": "2.3.0"}

@app.get("/health")
def health():
    return {"status": "ok"}
```

Pydantic валидирует вход: отрицательный `tenure_months` вернёт 422, а не мусорное предсказание.

### Dockerfile

```dockerfile
FROM python:3.12-slim
WORKDIR /srv
COPY requirements.lock .
RUN pip install --no-cache-dir -r requirements.lock
COPY app.py model/ ./
EXPOSE 8000
CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]
```

`requirements.lock` с точными версиями — иначе через полгода образ соберётся с другим sklearn и модель молча изменит предсказания.

### Мониторинг дрейфа (PSI)

```python
def psi(expected, actual, bins=10):
    """Population Stability Index: <0.1 ок, 0.1–0.25 насторожиться, >0.25 алерт"""
    cuts = np.quantile(expected, np.linspace(0, 1, bins + 1))
    e = np.histogram(expected, cuts)[0] / len(expected)
    a = np.histogram(actual,  cuts)[0] / len(actual)
    e, a = np.clip(e, 1e-4, None), np.clip(a, 1e-4, None)
    return np.sum((a - e) * np.log(a / e))

# еженедельный джоб: PSI по каждому признаку и по скорам модели
# неделя 1–6: все PSI < 0.08
# неделя 7:  days_since_last_login PSI = 0.31  ← АЛЕРТ
```

Расследование алерта: мобильное приложение выкатило автологин — «дней с последнего входа» массово обнулилось. Признак деградировал → плановое переобучение с новым определением активности. **Дрейф поймали до того, как ретеншен заметил падение качества списков.**

## 📌 Запомнить

- Модель в проде = код + артефакт + данные + версии, зафиксированные вместе.
- Валидация входа (Pydantic) — дёшево, спасает от мусорных предсказаний.
- Мониторинг: сервисные метрики + дрейф (PSI по признакам и скорам) + отложенное качество.
- Shadow → canary → раскатка; откат конфигом за минуту.
