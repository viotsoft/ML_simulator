## 📚 Теория

### Распределения — язык данных

**Нормальное распределение** — симметричный «колокол» (рост людей, ошибки измерений). **Скошенные распределения** — длинный хвост (доходы, суммы заказов: много мелких, мало огромных).

Практическое следствие: для скошенных данных **среднее обманывает**. Средний чек 3 400 ₽ при медиане 900 ₽ означает, что «типичный» заказ — 900 ₽, а среднее вытянуто китами. Докладывайте медиану и квантили.

### Проверка гипотез на пальцах

- **H₀ (нулевая гипотеза):** различия нет, разница между группами — случайность.
- **H₁ (альтернативная):** различие есть.
- **p-value** — вероятность увидеть такую (или более сильную) разницу, **если H₀ верна**. Малый p-value (обычно < 0.05) → данные плохо совместимы со «случайностью» → отвергаем H₀.

⚠️ Частые ошибки, за которые бьют на ревью:

1. p-value — это **не** «вероятность, что H₀ верна».
2. «Не значимо» ≠ «эффекта нет». Возможно, просто мало данных.
3. **Статистическая значимость ≠ практическая.** Рост конверсии на 0.01% может быть значимым при миллионах пользователей — и бесполезным для бизнеса.

### A/B-тест: минимальный чеклист

1. **Рандомизация:** пользователи распределяются случайно (не «A — Москва, B — Питер»!).
2. **Метрика решения выбрана до запуска** (например, прибыль на пользователя, не только конверсия).
3. **Размер выборки посчитан заранее** (power analysis), тест не останавливают в момент, когда «стало значимо» — это подглядывание (peeking) и оно ломает статистику.
4. Для конверсий (доля успехов) используется **z-тест для пропорций** или хи-квадрат; для средних значений — t-тест.

## 💻 Практика

### Шаг 1. Считаем значимость разницы конверсий

```python
import numpy as np
from statsmodels.stats.proportion import proportions_ztest

conversions = np.array([640, 580])   # успехи: A=6.4% и B=5.8% от 10 000
n_obs       = np.array([10000, 10000])

stat, p_value = proportions_ztest(conversions, n_obs)
print(f"p-value = {p_value:.4f}")    # p-value = 0.0721
```

p = 0.072 > 0.05 → при пороге 5% различие **не статистически значимо**. Данные совместимы с гипотезой «скидка не влияет».

### Шаг 2. Доверительный интервал эффекта

```python
diff = 0.064 - 0.058                          # +0.6 п.п.
se = np.sqrt(0.064*0.936/10000 + 0.058*0.942/10000)
ci = (diff - 1.96*se, diff + 1.96*se)
print(ci)                                     # (-0.0005, 0.0125)
```

Интервал накрывает ноль: истинный эффект может быть и нулевым, и до +1.25 п.п. Нужно больше данных.

### Шаг 3. Смотрим на деньги, а не только на конверсию

```python
# прибыль на пользователя (учитывая, что скидка режет маржу)

```

**Ответ Максу:**

1. Разница конверсий 6.4% vs 5.8% **статистически не доказана** (p = 0.07): с такой выборкой это могло быть случайностью. Чтобы надёжно отловить эффект +0.6 п.п., нужно ~30 000 пользователей на группу — можно продлить тест.
2. Даже если эффект реален, **прибыль на пользователя в группе со скидкой ниже** (41 ₽ против 49 ₽): скидка режет маржу сильнее, чем добавляет покупок. Это согласуется с выводом EDA из модуля 2.
3. Рекомендация: остановить массовые скидки; протестировать таргетированную скидку только «спящим» пользователям — там эффект может окупаться.

## 📌 Запомнить

- Медиана и квантили — для скошенных данных; среднее легко обманывает.
- p-value < 0.05 → отвергаем «случайность»; но значимость ≠ ценность для бизнеса.
- Метрику и размер выборки фиксируем **до** теста; peeking запрещён.
- Финальный аргумент — всегда деньги/целевая бизнес-метрика.
