## 📚 Theory

### What EDA is and why it matters

**EDA (Exploratory Data Analysis)** is the first thing any ML engineer does with a new dataset. Goals:

1. **Understand the structure:** which columns, types, how many rows.
2. **Find problems:** missing values, duplicates, outliers, type errors.
3. **See patterns:** distributions, correlations, trends.
4. **Form hypotheses** before building any models.

A rule from practice: **a model trained on unverified data is a bug you'll notice too late.**

### The essential Pandas toolkit

```python
import pandas as pd

df = pd.read_csv("sales.csv", parse_dates=["order_date"])

df.shape          # (50000, 9) — rows and columns
df.head()         # first 5 rows — look at the data with your eyes
df.info()         # column types and non-null counts
df.describe()     # numeric stats: mean, std, min, max, quartiles
df.isna().sum()   # missing values per column
df.duplicated().sum()  # number of full duplicates
df["category"].value_counts()  # category frequencies
```

### What to look for in `describe()`

- **Implausible min/max?** Price = −500 or age = 200 → data errors.
- **Mean far from the median (50%)?** The distribution is skewed, there are outliers.
- **std = 0?** A constant column, useless.

### Missing values: three strategies

| Strategy | When to use |
| --- | --- |
| Drop rows | < 1–2% missing and they are random |
| Fill (median/mode/special value) | noticeable share, the column matters |
| Keep as a feature | the very fact of missingness is informative (e.g. "didn't provide a phone number") |

### Outliers

The classic method is the **interquartile range (IQR)**: an outlier is a value outside `[Q1 − 1.5·IQR, Q3 + 1.5·IQR]`, where `IQR = Q3 − Q1`. But remember: **not every outlier is an error**. A 900,000 order may be a real wholesale purchase. Investigate outliers first, then decide.

## 💻 Practice

The `sales.csv` schema:

```
order_id, order_date, category, product_name, price, cost, quantity, discount_pct, city
```

### Step 1. Data quality

```python
df = pd.read_csv("sales.csv", parse_dates=["order_date"])
print(df.shape)                      # (50000, 9)
print(df.duplicated(subset="order_id").sum())   # 412 duplicates!
df = df.drop_duplicates(subset="order_id", keep="first")

print(df.isna().sum())
# discount_pct    3105  ← missing discounts
df["discount_pct"] = df["discount_pct"].fillna(0)  # no discount = 0, makes sense in this domain
```

The `order_id` duplicates are exactly the pipeline retries Anya mentioned. **Important:** look for duplicates by the business key (`order_id`), not across all columns.

### Step 2. Compute profit and look at the dynamics

```python
df["revenue"] = df["price"] * df["quantity"] * (1 - df["discount_pct"] / 100)
df["profit"]  = df["revenue"] - df["cost"] * df["quantity"]

monthly = df.set_index("order_date").resample("M")[["revenue", "profit"]].sum()
monthly.plot(title="Revenue and profit by month")
```

The picture: revenue grows from March, profit is flat.

### Step 3. Find the cause

```python
monthly_discount = df.set_index("order_date").resample("M")["discount_pct"].mean()
# March: 4.2% → June: 16.8%  ← the average discount grew 4x

df.groupby("category").agg(
    revenue=("revenue", "sum"),
    margin=("profit", lambda s: s.sum()),
).assign(margin_pct=lambda d: d.margin / d.revenue * 100)
# Electronics: margin dropped from 22% to 7%
```

### Step 4. Check the outliers

```python
q1, q3 = df["price"].quantile([0.25, 0.75])
iqr = q3 - q1
outliers = df[df["price"] > q3 + 1.5 * iqr]
# 89 orders, all in "Electronics", product "Dell Server" — real B2B purchases, NOT errors
```

**EDA report for Max (short version):**

1. **Data quality:** 412 duplicate orders (0.8%) from retries — removed; 3105 missing discounts — interpreted as "no discount". The data is fit for analysis.
2. **Key finding:** profit isn't growing because the revenue growth was bought with discounts. The average discount grew from 4.2% to 16.8%, and Electronics margin fell from 22% to 7%.
3. **Hypothesis:** the promos attract "discount hunters" who don't buy without promos. It can be tested with an A/B test (next module!).
4. **Not an error:** the expensive outlier orders are B2B purchases — treat them as a separate segment.

## 📌 The EDA checklist (save it)

1. `shape`, `head`, `info`, `describe` — always first.
2. Duplicates — by the business key.
3. Missing values — understand *why* they exist, then decide what to do.
4. Outliers — investigate, don't blindly delete.
5. Dates — convert to datetime, check the range and gaps.
6. Conclusions — in business language, not "p-values and quartiles".
