## 📚 Theory

### Why not a fully connected network

A 224×224×3 photo = 150,528 inputs. A dense layer of 1000 neurons — 150M weights, and the net would have to learn a "crumpled corner" separately at every position in the frame. **Convolution** solves both problems: a small filter (e.g. 3×3) slides across the image and looks for the same pattern everywhere (weight sharing).

### How a CNN works

- **Convolutional layers** — learn a feature hierarchy: early layers see edges and corners, middle ones — textures, deep ones — "crumpledness", "tears".
- **Pooling** — reduces resolution, gives shift tolerance.
- **The head** — a dense layer on top of the features → classes.

### Transfer learning — practical CV's main tool

Networks pre-trained on ImageNet (millions of images) already know how to see edges, textures, shapes. Take a pre-trained backbone (ResNet, EfficientNet), then two regimes:

1. **Feature extraction:** freeze the backbone, train only the new head. For tiny datasets.
2. **Fine-tuning:** unfreeze some/all layers and retrain with a small LR (10–100× smaller than usual) so as not to destroy the pre-trained weights.

### Augmentations — free data

Random rotations, flips, brightness/contrast, crops — from 4,000 photos we make an "infinite" stream of variations. Rules: augmentations — **train only**; choose realistic ones (a box is never upside down on the conveyor? — no vertical flip then).

## 💻 Practice

```python
import torch, torch.nn as nn
from torchvision import models, transforms
from torchvision.datasets import ImageFolder

train_tf = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.RandomHorizontalFlip(),
    transforms.ColorJitter(brightness=0.3, contrast=0.3),
    transforms.RandomRotation(10),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),  # ImageNet statistics
])

model = models.resnet18(weights="IMAGENET1K_V1")
model.fc = nn.Linear(model.fc.in_features, 2)     # our own head: ok / defect

# stage 1: head only (backbone frozen)
for p in model.parameters(): p.requires_grad = False
for p in model.fc.parameters(): p.requires_grad = True
opt = torch.optim.AdamW(model.fc.parameters(), lr=1e-3)
# ... 5 epochs → val recall(defect) = 0.71

# stage 2: fine-tune everything with a small LR
for p in model.parameters(): p.requires_grad = True
opt = torch.optim.AdamW(model.parameters(), lr=1e-5)
class_weights = torch.tensor([1.0, 9.0])          # defects are 9× rarer
loss_fn = nn.CrossEntropyLoss(weight=class_weights)
# ... 10 epochs → val recall(defect) = 0.93 at precision 0.78
```

### Evaluation at the operating point

Error costs: a missed defect = a return (−$8 + an angry customer); a false alert = 10 seconds of the inspector's time. So we shift the threshold toward recall: at 0.35, recall = 0.96, precision = 0.61 — the inspector checks ~50 boxes an hour instead of all of them, missing 4% of defects instead of 100% without the system.

<details>
<summary><b>Review with Lena</b></summary>

1. **The two-stage scheme** (head → full fine-tuning with a small LR) is the standard: first "dock" the head, then gently adapt the features.
2. `Normalize` with ImageNet statistics is mandatory — the model was pre-trained on such inputs.
3. The idiot-proof check: we looked at Grad-CAM (attention heatmaps) — the model looks at the damage, not the conveyor belt or the barcode. There was an industry case where a "defect detector" learned that defects were photographed by a different camera.
4. Dataset drift ahead: winter lighting in the workshop is different — we agreed on a monthly re-labeling of 200 photos and monitoring of the alert share (a bridge to module 19).
</details>

## 📌 Remember

- Convolutions search for one pattern across the whole frame — hence CNNs, not dense nets.
- Small dataset → transfer learning: the head first, then fine-tuning with a small LR.
- Augmentations — train only, and realistic only.
- The threshold — from error costs; check **where** the model looks (Grad-CAM).
