## 📚 Theory

### Text → numbers: the evolution of approaches

1. **Bag of Words:** text = a vector of word counts. Loses word order.
2. **TF-IDF:** counts weighted by corpus rarity. A word frequent in a document but rare in the corpus ("refund") matters more than one frequent everywhere ("hello"). Still the strongest classification baseline.
3. **Word embeddings (word2vec, fastText):** word → dense vector; semantically similar words are close. fastText learns vectors from character n-grams — robust to typos.
4. **Transformers (BERT and heirs):** a word's vector depends on **context** ("the card is blocked" vs "a card of the city"). Pre-trained on huge corpora, fine-tuned for the task.

### The classic NLP pipeline

- **Tokenization** — split into words/tokens.
- **Normalization:** lowercase; lemmatization (critical for morphologically rich languages: "payments", "payment's", "paid" → the lemma).
- **N-grams:** word pairs catch "not working", "money not received" — negations change the meaning.

### When to use what

| Approach | Pros | Cons |
| --- | --- | --- |
| TF-IDF + linear model | fast, cheap, interpretable, CPU | doesn't understand context or synonyms |
| Fine-tuned transformer | best quality, robust to phrasing | GPU, latency, harder maintenance |

The decision, as always, comes from the requirements: latency, budget, size of the win.

## 💻 Practice

### Baseline: TF-IDF + logreg

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline
from sklearn.metrics import f1_score, classification_report

pipe = Pipeline([
    ("tfidf", TfidfVectorizer(lowercase=True, ngram_range=(1, 2),
                              max_features=100_000, min_df=3)),
    ("clf", LogisticRegression(max_iter=1000, C=5.0)),
])
pipe.fit(train.text, train.category)
pred = pipe.predict(test.text)
print(f1_score(test.category, pred, average="macro"))   # macro-F1: 0.86
```

macro-F1 (the average over classes) — because classes differ in size, and the rare "fraud" category matters more than the rest.

### Transformer: fine-tuning

```python
from transformers import (AutoTokenizer, AutoModelForSequenceClassification,
                          TrainingArguments, Trainer)

model_name = "distilbert-base-uncased"       # a compact BERT
tok = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=5)

def encode(batch): return tok(batch["text"], truncation=True, max_length=256)
trainer = Trainer(model=model,
                  args=TrainingArguments("out", num_train_epochs=3,
                                         per_device_train_batch_size=32,
                                         eval_strategy="epoch"),
                  train_dataset=ds_train.map(encode, batched=True),
                  eval_dataset=ds_val.map(encode, batched=True))
trainer.train()
# macro-F1: 0.91
```

### The comparison for the decision

| | macro-F1 | latency (CPU) | hardware |
| --- | --- | --- | --- |
| TF-IDF + logreg | 0.86 | 2 ms | CPU |
| DistilBERT (fine-tuned) | 0.91 | 40 ms | CPU ok |
| large BERT | 0.925 | 300 ms | GPU |

<details>
<summary><b>The decision and the review</b></summary>

**Choice: DistilBERT.** +5 pp F1 over the baseline is ~250 tickets a day landing in the right queue immediately; 40 ms latency for asynchronous sorting is a non-issue. Large BERT adds another +1.5 pp but needs a GPU — doesn't pay off.

Production details:
1. **Confidence threshold:** if max softmax < 0.6 — the ticket goes to manual sorting. We automate the confident 85%, not everything.
2. We analyze errors with a per-class confusion matrix: "payments" get confused with "fraud" — we added key features (presence of an amount, phrases like "charged twice").
3. The TF-IDF baseline stays as a fallback and a sanity check for new versions.
4. Further down this road — an LLM with few-shot for new categories without labels, but that's a different budget.
</details>

## 📌 Remember

- TF-IDF + a linear model is the mandatory NLP classification baseline.
- Lemmatization and n-grams help a lot with morphologically rich languages.
- Transformers add context; choose the model size from latency requirements.
- Confidence threshold + a manual queue = safe automation.
