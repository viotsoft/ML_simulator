## 📚 Theory

### The decision tree

A tree splits the data with a series of "feature > threshold?" questions, at each step choosing the split that maximally reduces "impurity" (Gini/entropy). Pros: needs no scaling, captures non-linearities and feature interactions, interpretable. Con: **a single tree overfits easily** — a deep tree memorizes noise.

### Ensembles: the wisdom of the crowd

The idea: many weak models that err **differently** are together more accurate than one strong model.

**Bagging → Random Forest:**
- Each tree trains on a random row subsample (bootstrap) and a random feature subset.
- Predictions are averaged → variance drops, overfitting is smoothed out.
- Trees are independent → training parallelizes.

**Boosting → XGBoost / LightGBM / CatBoost:**
- Trees are built **sequentially**: each next one corrects the errors (fits the gradient of the loss) of the previous ones.
- Trees are shallow (3–8 levels), and there are hundreds or thousands of them.
- Usually more accurate than a forest, but more sensitive to hyperparameters and easier to overfit.

| | Random Forest | Gradient Boosting |
| --- | --- | --- |
| Strategy | parallel, averaging | sequential, error correction |
| Overfitting | robust | needs control (early stopping) |
| Training speed | faster (parallel) | slower, but LightGBM is very fast |
| Tabular quality | good | usually the best |

### Key boosting hyperparameters

- `n_estimators` — number of trees; with **early stopping** you can set it high.
- `learning_rate` — each tree's contribution; lower = more stable, but needs more trees.
- `max_depth` / `num_leaves` — tree complexity, the main overfitting knob.
- `early_stopping_rounds` — stop when the validation metric hasn't improved for N iterations.

### Why boosting > neural nets on tables

Tabular data is heterogeneous (money, counters, categories) and the dependencies are piecewise-sharp ("delinquency > 30 days → risk jumps"). Trees model such thresholds natively; neural networks struggle. Confirmed by industry and Kaggle: **for tables the default is gradient boosting.**

## 💻 Practice

```python
from lightgbm import LGBMClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import roc_auc_score

# the same churn sample as module 5, plus the module-7 features
rf = RandomForestClassifier(n_estimators=500, max_depth=None,
                            min_samples_leaf=20, n_jobs=-1, random_state=42)
rf.fit(X_train, y_train)
print(f"RF  ROC-AUC: {roc_auc_score(y_test, rf.predict_proba(X_test)[:,1]):.3f}")   # 0.861

lgbm = LGBMClassifier(n_estimators=3000, learning_rate=0.03,
                      num_leaves=63, random_state=42)
lgbm.fit(X_train, y_train,
         eval_set=[(X_val, y_val)],
         eval_metric="auc",
         callbacks=[lightgbm.early_stopping(100)])    # stop on validation
print(f"LGBM ROC-AUC: {roc_auc_score(y_test, lgbm.predict_proba(X_test)[:,1]):.3f}") # 0.874
```

Result: logreg 0.834 → RF 0.861 → LightGBM **0.874**.

## 📌 Remember

- A single tree overfits; the power is in ensembles.
- Bagging reduces variance by averaging; boosting sequentially corrects errors.
- Early stopping on validation is the standard for boosting.
- For tabular data gradient boosting is the industry default.
