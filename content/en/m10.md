## 📚 Theory

### Parameters vs hyperparameters

- **Parameters** the model learns itself (weights, tree split thresholds).
- **Hyperparameters** the engineer sets before training (tree depth, learning rate, regularization strength).

### Three search strategies

| Strategy | How it works | When |
| --- | --- | --- |
| **Grid search** | full sweep of a grid | 2–3 parameters with few values |
| **Random search** | random points from distributions | the default choice: for the same budget it covers the space better (Bergstra & Bengio, 2012) |
| **Bayesian (Optuna, TPE)** | builds a "parameters → quality" model and tries promising points | expensive training, many parameters — the industry standard |

Why random > grid: if only 1 of 3 parameters matters, a 5×5×5 grid tries just 5 unique values of the important one; random tries all 125.

### What to tune in gradient boosting (in order of importance)

1. `learning_rate` + `n_estimators` (with early stopping — fix a lowish LR, plenty of trees);
2. `num_leaves` / `max_depth` — model capacity;
3. `min_child_samples` — leaf regularization;
4. `feature_fraction`, `bagging_fraction` — stochasticity;
5. `lambda_l1`, `lambda_l2` — if it still overfits.

### Overfitting to validation — the silent killer

Hundreds of tuning iterations against a single validation set = you "fit" the model to it. Rules:

- Evaluate candidates with **cross-validation**, not a single split.
- Measure the final number on a **test set that took no part in tuning, exactly once**.
- A gain smaller than the CV spread (std) is noise, not improvement.

## 💻 Practice: Optuna

```python
import optuna
from lightgbm import LGBMClassifier
from sklearn.model_selection import cross_val_score, StratifiedKFold

def objective(trial):
    params = {
        "learning_rate": trial.suggest_float("learning_rate", 0.01, 0.2, log=True),
        "num_leaves": trial.suggest_int("num_leaves", 15, 255, log=True),
        "min_child_samples": trial.suggest_int("min_child_samples", 5, 200, log=True),
        "feature_fraction": trial.suggest_float("feature_fraction", 0.5, 1.0),
        "bagging_fraction": trial.suggest_float("bagging_fraction", 0.5, 1.0),
        "lambda_l2": trial.suggest_float("lambda_l2", 1e-3, 10, log=True),
        "n_estimators": 2000,
    }
    model = LGBMClassifier(**params, random_state=42)
    cv = StratifiedKFold(5, shuffle=True, random_state=42)
    return cross_val_score(model, X_train, y_train, cv=cv,
                           scoring="roc_auc", n_jobs=-1).mean()

study = optuna.create_study(direction="maximize",
                            sampler=optuna.samplers.TPESampler(seed=42))
study.optimize(objective, n_trials=60, timeout=3600)   # an hour — and that's enough

print(study.best_value)    # CV AUC: 0.8815 (was 0.8742 on defaults)
print(study.best_params)
```

### The final check

```python
best = LGBMClassifier(**study.best_params, n_estimators=2000, random_state=42)
best.fit(X_train, y_train)
print(roc_auc_score(y_test, best.predict_proba(X_test)[:, 1]))
# Test: 0.879 (was 0.874). CV std was 0.004 → the 0.005 gain is borderline noise, but stable across folds.
```

Optuna niceties: `study.trials_dataframe()` — the log of all trials; `optuna.visualization.plot_param_importances(study)` — which parameters actually mattered (often 80% of the effect comes from learning_rate + num_leaves).

<details>
<summary><b>Review with Lena</b></summary>

1. **Budget first:** 60 trials / 1 hour is a conscious constraint. Tuning has diminishing returns; after the first ~50 trials TPE rarely finds substantially better.
2. **log=True** for the learning rate and regularization — search for the "order of magnitude", not linear steps.
3. A +0.005 AUC gain with std 0.004 is honestly "borderline noise". A real boost would come from new features (module 7) — tuning never replaces data.
4. The classic effort priority: **data > features > model > hyperparameters.**
</details>

## 📌 Remember

- Random search beats grid search almost always; Optuna is the standard for expensive training.
- Tune with CV, measure once on an untouched test.
- A gain smaller than the CV std is noise.
- Tuning is the last optimization step, not the first.

**Further reading:** Bergstra & Bengio, "Random Search for Hyper-Parameter Optimization"; the Optuna docs.
