## 📚 Theory

### Distributions — the language of data

The **normal distribution** is a symmetric "bell" (human height, measurement errors). **Skewed distributions** have a long tail (incomes, order values: many small ones, few huge ones).

Practical consequence: for skewed data **the mean lies**. An average order of $45 with a median of $12 means the "typical" order is $12, and the mean is pulled up by whales. Report the median and quantiles.

### Hypothesis testing in plain words

- **H₀ (null hypothesis):** there is no difference; the gap between groups is chance.
- **H₁ (alternative):** there is a difference.
- **p-value** — the probability of seeing such a difference (or larger) **if H₀ is true**. Small p-value (usually < 0.05) → the data is poorly compatible with "chance" → reject H₀.

⚠️ Frequent mistakes that get you flagged in review:

1. The p-value is **not** "the probability that H₀ is true".
2. "Not significant" ≠ "no effect". Maybe there's just not enough data.
3. **Statistical significance ≠ practical significance.** A 0.01% conversion lift can be significant with millions of users — and useless for the business.

### A/B test: the minimal checklist

1. **Randomization:** users are assigned randomly (not "A — Moscow, B — St. Petersburg"!).
2. **The decision metric is chosen before launch** (e.g. profit per user, not just conversion).
3. **Sample size is computed in advance** (power analysis); the test is not stopped the moment it "became significant" — that's peeking and it breaks the statistics.
4. For conversions (success proportions) use a **z-test for proportions** or chi-squared; for means — a t-test.

## 💻 Practice

### Step 1. Test the significance of the conversion difference

```python
import numpy as np
from statsmodels.stats.proportion import proportions_ztest

conversions = np.array([640, 580])   # successes: A=6.4% and B=5.8% of 10,000
n_obs       = np.array([10000, 10000])

stat, p_value = proportions_ztest(conversions, n_obs)
print(f"p-value = {p_value:.4f}")    # p-value = 0.0721
```

p = 0.072 > 0.05 → at the 5% threshold the difference is **not statistically significant**. The data is compatible with "the discount has no effect".

### Step 2. Confidence interval of the effect

```python
diff = 0.064 - 0.058                          # +0.6 pp
se = np.sqrt(0.064*0.936/10000 + 0.058*0.942/10000)
ci = (diff - 1.96*se, diff + 1.96*se)
print(ci)                                     # (-0.0005, 0.0125)
```

The interval covers zero: the true effect could be zero or up to +1.25 pp. We need more data.

### Step 3. Look at the money, not just conversion

```python
# profit per user (accounting for the discount cutting the margin)
profit_A = 640 * avg_check * margin_with_discount / 10000
profit_B = 580 * avg_check * margin_base / 10000
# A: $0.52/user, B: $0.61/user  ← the NO-discount group is more profitable!
```

**Answer for Max:**

1. The 6.4% vs 5.8% conversion difference is **not statistically proven** (p = 0.07): with this sample it could be chance. To reliably detect a +0.6 pp effect we need ~30,000 users per group — the test can be extended.
2. Even if the effect is real, **profit per user in the discount group is lower** ($0.52 vs $0.61): the discount cuts the margin more than it adds purchases. This matches the EDA conclusion from module 2.
3. Recommendation: stop mass discounts; test a targeted discount only for "dormant" users — there the effect may pay off.

## 📌 Remember

- Median and quantiles for skewed data; the mean is easily misleading.
- p-value < 0.05 → reject "chance"; but significance ≠ business value.
- Fix the metric and sample size **before** the test; peeking is forbidden.
- The final argument is always money / the target business metric.
