## 📚 Теория

### Почему не полносвязная сеть

Фото 224×224×3 = 150 528 входов. Полносвязный слой на 1000 нейронов — 150 млн весов, и сеть должна отдельно выучить «мятый угол» в каждой точке кадра. **Свёртка** решает обе проблемы: маленький фильтр (например 3×3) скользит по всему изображению и ищет один и тот же паттерн везде (weight sharing). 

### Как устроена CNN

- **Свёрточные слои** — учат иерархию признаков: первые слои видят края и углы, средние — текстуры, глубокие — «мятость», «разрыв».
- **Pooling** — уменьшает разрешение, даёт устойчивость к сдвигам.
- **Голова** — полносвязный слой поверх признаков → классы.

### Transfer learning — главный приём практического CV

Сети, предобученные на ImageNet (миллионы картинок), уже умеют видеть края, текстуры, формы. Берём предобученный backbone (ResNet, EfficientNet), а дальше два режима:

1. **Feature extraction:** замораживаем backbone, учим только новую голову. Для крошечных датасетов.
2. **Fine-tuning:** размораживаем часть/все слои и дообучаем с маленьким LR (10–100× меньше обычного), чтобы не разрушить предобученные веса.

### Аугментации — бесплатные данные

Случайные повороты, отражения, яркость/контраст, кропы — из 4 000 фото делаем «бесконечный» поток вариаций. Правила: аугментации — **только на train**; выбирать реалистичные (коробка не бывает вверх ногами на конвейере? — вертикальный флип не нужен).

## 💻 Практика

```python
import torch, torch.nn as nn
from torchvision import models, transforms
from torchvision.datasets import ImageFolder

train_tf = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.RandomHorizontalFlip(),
    transforms.ColorJitter(brightness=0.3, contrast=0.3),
    transforms.RandomRotation(10),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),  # статистики ImageNet
])

model = models.resnet18(weights="IMAGENET1K_V1")
model.fc = nn.Linear(model.fc.in_features, 2)     # своя голова: ok / defect

# этап 1: только голова (backbone заморожен)
for p in model.parameters(): p.requires_grad = False
for p in model.fc.parameters(): p.requires_grad = True
opt = torch.optim.AdamW(model.fc.parameters(), lr=1e-3)
# ... 5 эпох → val recall(defect) = 0.71

# этап 2: fine-tuning всего с малым LR
for p in model.parameters(): p.requires_grad = True
opt = torch.optim.AdamW(model.parameters(), lr=1e-5)
class_weights = torch.tensor([1.0, 9.0])          # брак в 9 раз реже
loss_fn = nn.CrossEntropyLoss(weight=class_weights)
# ... 10 эпох → val recall(defect) = 0.93 при precision 0.78
```

### Оценка в рабочей точке

Цена ошибок: пропущенный брак = возврат (−600 ₽ + злой клиент), ложный алерт = 10 секунд контролёра. Значит, смещаем порог в сторону recall: при пороге 0.35 recall = 0.96, precision = 0.61 — контролёр проверяет ~50 коробок в час вместо всех, пропуская 4% брака вместо 100% пропусков без системы.

<details>
<summary><b>Разбор с Леной</b></summary>

1. **Двухэтапная схема** (голова → full fine-tuning с малым LR) — стандарт: сначала «пристыковали» голову, потом аккуратно адаптировали признаки.
2. `Normalize` со статистиками ImageNet обязателен — модель предобучена на таких входах.
3. Проверка на дурака: посмотрели Grad-CAM (тепловые карты внимания) — модель смотрит на повреждения, а не на конвейерную ленту или штрихкод. Был случай в индустрии, когда «детектор брака» выучил, что брак снимали на другую камеру.
4. Датасет-дрейф впереди: зимой освещение в цехе другое — договорились о ежемесячной доразметке 200 фото и мониторинге доли алертов (мостик к модулю 19).

## 📌 Запомнить

- Свёртки ищут один паттерн по всему кадру — поэтому CNN, а не полносвязные сети.
- Маленький датасет → transfer learning: голова, потом fine-tuning с малым LR.
- Аугментации — только train и только реалистичные.
- Порог — из цены ошибок; проверяйте, **куда** смотрит модель (Grad-CAM).
