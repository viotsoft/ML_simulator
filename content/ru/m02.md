## 📚 Теория

### Что такое EDA и зачем он нужен

**EDA (Exploratory Data Analysis)** — разведочный анализ данных. Это первое, что делает любой ML-инженер с новым датасетом. Цели:

1. **Понять структуру:** какие колонки, типы, сколько строк.
2. **Найти проблемы:** пропуски, дубликаты, выбросы, ошибки типов.
3. **Увидеть закономерности:** распределения, корреляции, тренды.
4. **Сформулировать гипотезы** до построения моделей.

Правило из практики: **модель, обученная на непроверенных данных, — это баг, который вы заметите слишком поздно.**

### Джентльменский набор Pandas

```python
import pandas as pd

df = pd.read_csv("sales.csv", parse_dates=["order_date"])

df.shape          # (50000, 9) — строки и колонки
df.head()         # первые 5 строк — глазами посмотреть на данные
df.info()         # типы колонок и число непустых значений
df.describe()     # статистики числовых колонок: mean, std, min, max, квартили
df.isna().sum()   # пропуски по колонкам
df.duplicated().sum()  # число полных дубликатов
df["category"].value_counts()  # частоты категорий
```

### На что смотреть в `describe()`

- **min/max неправдоподобны?** Цена = −500 или возраст = 200 → ошибки данных.
- **mean сильно отличается от медианы (50%)?** Распределение скошено, есть выбросы.
- **std = 0?** Колонка-константа, бесполезна.

### Пропуски: три стратегии

| Стратегия | Когда применять |
| --- | --- |
| Удалить строки | пропусков < 1–2% и они случайны |
| Заполнить (медиана/мода/спец-значение) | пропусков заметно, колонка важна |
| Оставить как признак | сам факт пропуска информативен (например, «не указал телефон») |

### Выбросы

Классический метод — **межквартильный размах (IQR)**: выброс — это значение вне `[Q1 − 1.5·IQR, Q3 + 1.5·IQR]`, где `IQR = Q3 − Q1`. Но помните: **не каждый выброс — ошибка**. Заказ на 900 000 ₽ может быть реальной оптовой закупкой. Выбросы сначала исследуют, потом решают.

## 💻 Практика

Схема `sales.csv`:

```
order_id, order_date, category, product_name, price, cost, quantity, discount_pct, city
```

### Шаг 1. Качество данных

```python
df = pd.read_csv("sales.csv", parse_dates=["order_date"])
print(df.shape)                      # (50000, 9)
print(df.duplicated(subset="order_id").sum())   # 412 дублей!
df = df.drop_duplicates(subset="order_id", keep="first")

print(df.isna().sum())
# discount_pct    3105  ← пропуски в скидке
df["discount_pct"] = df["discount_pct"].fillna(0)  # нет скидки = 0, логично для домена
```

Дубли по `order_id` — те самые ретраи пайплайна, о которых говорила Аня. **Важно:** дубли ищем по бизнес-ключу (`order_id`), а не по всем колонкам.

### Шаг 2. Считаем прибыль и смотрим динамику

```python
df["revenue"] = df["price"] * df["quantity"] * (1 - df["discount_pct"] / 100)
df["profit"]  = df["revenue"] - df["cost"] * df["quantity"]

monthly = df.set_index("order_date").resample("M")[["revenue", "profit"]].sum()
monthly.plot(title="Выручка и прибыль по месяцам")
```

Картина: выручка растёт с марта, прибыль — плоская.

### Шаг 3. Ищем причину

```python
monthly_discount = df.set_index("order_date").resample("M")["discount_pct"].mean()
# март: 4.2% → июнь: 16.8%  ← средняя скидка выросла в 4 раза

df.groupby("category").agg(
    revenue=("revenue", "sum"),
    margin=("profit", lambda s: s.sum()),
).assign(margin_pct=lambda d: d.margin / d.revenue * 100)
# Электроника: маржа упала с 22% до 7%
```

### Шаг 4. Проверяем выбросы

```python
q1, q3 = df["price"].quantile([0.25, 0.75])
iqr = q3 - q1
outliers = df[df["price"] > q3 + 1.5 * iqr]
# 89 заказов, все — категория "Электроника", товар "Сервер Dell" — реальные B2B-закупки, НЕ ошибки
```

**EDA-отчёт для Макса (кратко):**

1. **Качество данных:** 412 дублей заказов (0.8%) из-за ретраев — удалены; 3105 пропусков в скидке — интерпретированы как «без скидки». Данные пригодны для анализа.
2. **Главный вывод:** прибыль не растёт, потому что рост выручки куплен скидками. Средняя скидка выросла с 4.2% до 16.8%, маржа «Электроники» упала с 22% до 7%.
3. **Гипотеза:** промо-кампании привлекают «охотников за скидками», которые не покупают без промо. Проверить можно A/B-тестом (следующий модуль!).
4. **Не ошибка:** дорогие заказы-выбросы — это B2B-закупки, их надо учитывать отдельным сегментом.

## 📌 Чеклист EDA (сохраните себе)

1. `shape`, `head`, `info`, `describe` — всегда первым делом.
2. Дубликаты — по бизнес-ключу.
3. Пропуски — понять *почему* они есть, потом решать что делать.
4. Выбросы — исследовать, а не слепо удалять.
5. Даты — привести к datetime, проверить диапазон и дыры.
6. Вывод — на языке бизнеса, а не «p-value и квартили».
