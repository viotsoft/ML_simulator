# ML Career Simulator

Игровой симулятор карьеры ML-инженера: пользователь «устраивается на работу» джуниором в вымышленную компанию **Datacore** и за 20 модулей вырастает до Middle ML Engineer, решая реальные бизнес-кейсы.

## Запуск локально

```bash
npm install
npm start
# → http://localhost:3000
```

- `http://localhost:3000` — лендинг с описанием продукта и тарифами.
- `http://localhost:3000/app.html` — приложение: регистрация, модули, квизы, сертификат.

## Как это устроено

| Часть | Файлы | Описание |
| --- | --- | --- |
| Бэкенд | `server.js` | Express: auth (scrypt + cookie-сессии), API модулей/квизов/подписки/собеседований/сертификата |
| Контент | `content/m01.md … m23.md` | 20 базовых модулей + 3 Advanced-модуля по книге «ML System Design» (Бабушкин, Кравченко); в каждом — SVG-визуализации теории |
| Манифест | `content/modules.json` | Порядок, названия, уровни модулей |
| Квизы | `content/quizzes.json` | 120 вопросов с правильными ответами и объяснениями |
| Собеседования | `content/interviews.json` | 3 трека: Junior (12 Q&A), Middle (12 Q&A), ML System Design (8 полных разборов) |
| Фронтенд | `public/` | Лендинг (`index.html`) + SPA (`app.html`, `js/app.js`, `css/style.css`) |
| Данные | `data/users.json` | Создаётся автоматически: пользователи, сессии, прогресс |

## Разделы приложения

- **📚 Модули** — курс: модули 1–20 (Junior → Middle, сертификат) + 21–23 (Advanced-трек по ML System Design, бонус после сертификата).
- **🎤 Собеседования** — тренажёр интервью: вопрос → ответить вслух → открыть эталонный разбор. Junior-трек бесплатный, Middle и System Design — по подписке.
- Сертификат выдаётся за 20 базовых модулей (константа `CERT_REQUIRED` в `server.js`).

## Бизнес-модель

- Модули 1–2 и Junior-трек собеседований — **бесплатно** (регистрация без карты).
- Модули 3–23 и треки Middle / System Design — подписка **$20/мес** (кнопка «Оформить подписку»).
- Сертификат «Middle-track ML Engineer» — после прохождения квизов модулей 1–20 (порог 70%).

## Оплата: WayForPay + Stripe

Приложение умеет принимать платежи через **два** шлюза; активный режим выбирается
переменными окружения (см. `.env.example`), приоритет: WayForPay → Stripe → демо.

### WayForPay (для мерчанта из Украины, карты со всего мира)

Stripe и PayPal не работают как платёжные шлюзы для мерчанта из России/Украины —
у WayForPay такого ограничения нет, а принимать он умеет международные карты.

- `WFP_MERCHANT_ACCOUNT` + `WFP_MERCHANT_SECRET` + `WFP_DOMAIN` → кнопка подписки
  открывает **встроенный виджет оплаты** WayForPay прямо на странице (без ухода на
  внешний сайт), с автосписанием раз в месяц (`regularMode: monthly`);
- `/api/wayforpay/webhook` принимает уведомления о каждом платеже (первом и всех
  последующих автосписаниях), проверяет подпись HMAC-MD5 и активирует подписку;
- `WFP_MERCHANT_PASSWORD` — отдельный секрет для отмены подписки через их API
  управления регулярными платежами (кнопка «Подписка» в шапке → `/api/wayforpay/cancel`);
- **без ключей — демо-режим**, подписка включается мгновенно без оплаты.

Реализация проверена: подпись сверена независимым пересчётом, публичный тестовый
аккаунт WayForPay (`test_merch_n1`) подтверждает и открывает виджет с правильными
данными заказа. Подробности, реквизиты песочницы и как получить боевой аккаунт —
в [DEPLOY.md](DEPLOY.md).

### Stripe (если принимаете платежи не из Украины/РФ/СНГ)

- `STRIPE_SECRET_KEY` + `STRIPE_PRICE_ID` → кнопка подписки ведёт на **Stripe Checkout**;
- `/api/stripe/webhook` обрабатывает `checkout.session.completed` (активация),
  `customer.subscription.updated/deleted` (продление/отмена) — задайте `STRIPE_WEBHOOK_SECRET`;
- после возврата с оплаты `/api/checkout/confirm` активирует подписку, не дожидаясь вебхука;
- кнопка «Подписка» в шапке открывает **Stripe Billing Portal** (смена карты, отмена).

Пошаговая настройка обеих платёжек и деплой — в [DEPLOY.md](DEPLOY.md).

## Маркетинг: автопостинг в соцсети

Встроенный SMM-конвейер с веб-панелью **`/admin/marketing`**: генерация EN-контента
из материалов курса (Claude API), карточки и видео с музыкой, автопостинг по
расписанию в Facebook, Instagram, LinkedIn, TikTok, Threads и X, OAuth-подключение
сетей кнопками, utm-атрибуция регистраций до конкретного поста. Движок — в
`marketing/`, подробности — в [MARKETING.md](MARKETING.md).

## Админка: кто зарегистрировался

`/admin` — список всех пользователей (имя, email, дата регистрации, PRO/FREE, прогресс,
сертификат). Защищена HTTP Basic Auth по переменной `ADMIN_PASSWORD` (логин любой, значение —
пароль); без этой переменной раздел выключен целиком (404). Хэши и соли паролей в ответ
API `/api/admin/users` не попадают.

Данные пользователей до сих пор хранятся не в СУБД, а в одном JSON-файле
(`data/users.json` → на хостинге путь из `DATA_DIR`, у нас — постоянный диск Railway).
Годится для MVP; при заметном росте регистраций стоит перейти на SQLite/Postgres —
файл легко читает гонки записи при параллельных запросах.

## Настройки (server.js)

- `FREE_MODULES = 2` — сколько первых модулей бесплатны.
- `PASS_SCORE = 0.7` — порог прохождения квиза.
- `PORT` — переменная окружения (по умолчанию 3000).

## Добавление нового модуля

1. Создайте `content/m21.md` (структура: ситуация → задача → теория → практика → разбор).
2. Добавьте запись в `content/modules.json`.
3. Добавьте вопросы в `content/quizzes.json` под ключом `m21`.
4. Перезапустите сервер.

## Что стоит сделать перед реальным продом

- Заменить JSON-хранилище на PostgreSQL/SQLite.
- Stripe для подписок (см. выше), email-подтверждение регистрации.
- HTTPS + флаг `Secure` у cookie, rate limiting на auth-эндпоинты.
- Хостинг: любой Node-хостинг (Railway, Render, Fly.io) — приложение без нативных зависимостей.
